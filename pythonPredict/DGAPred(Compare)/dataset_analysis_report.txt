DGAPred(Compare) 数据集分析报告
生成时间: 2026-06-18

一、分析目的
本报告用于观察当前 DGAPred(Compare) 训练数据集的结构特征、潜在风险点，以及哪些数据集优化方式更可能带来 AUC/AUPR/ACC/MCC 提升。

说明:
1. 本报告先做“观察”和“策略建议”，不直接修改训练代码或评估逻辑。
2. 目标是找出“合理清洗”和“刷分删样本”的边界，避免为了分数好看而破坏任务真实性。


二、当前数据集现况
1. 标签矩阵来源
- 训练入口在 src/main.py 的 load_label() 中。
- 原始正样本文件: 2drug-2side/DGAPred/data/sider_pert_mesh_list.csv
- 该文件中全部都是正样本，shape = (29233, 3)
- 列为: pert_id, MESH_ID, label
- 原始文件中:
  - unique pert_id = 546
  - unique MESH_ID = 1233

2. 经过当前筛选后的主标签矩阵
- 当前实际训练使用的是 pythonPredict/drug_side.csv
- shape = (453, 1019)
- 正样本数 = 23395
- 负样本数 = 438212
- 正样本占比 = 0.05068164044306087

3. 重要结论
- 当前任务是明显的稀疏二分类任务。
- 原始文件只给出“已知正样本”。
- 训练中大量负样本不是人工确认的真负样本，而是“矩阵中未观察到的 0”。
- 因此当前数据集天然存在“伪负样本”风险，这类样本会直接拖低 AUC 和 AUPR。

3. 这类分布为什么会拖指标
- 正样本过稀时，模型很难学到稳定模式。
- 单例和双例实体对测试划分非常敏感，某一折里一旦被分到测试侧，就更容易变成难样本。
- 这类样本通常更容易拉低 AUC、AUPR 和 MCC。


四、负样本构造风险
1. 当前负样本的本质
- 当前训练数据中的负样本主要来自 drug_side 矩阵中值为 0 的位置。
- 这些 0 并不等于“明确无关联”，更像“尚未观察到关联”。

2. 伪负样本风险
- 如果某个 drug 与另一个高相似 drug 非常接近，而后者已经关联多个 ADR，
  那么前者对应的未标注 0 很可能并不是真负样本。
- 这种“应该接近正样本、却被当成负样本”的点，会显著拖累 AUC。

3. 粗观察到的高风险 drug
以下 drug 周围存在较多“高相似邻居已知为正、自己却未标注”的候选难负样本:
- BRD-A74980173: 相似 drug 数 2, 自身正样本 32, 候选难负样本 117
- BRD-A62025033: 相似 drug 数 1, 自身正样本 21, 候选难负样本 106
- BRD-K11196887: 相似 drug 数 2, 自身正样本 71, 候选难负样本 105
- BRD-A67981824: 相似 drug 数 4, 自身正样本 20, 候选难负样本 104
- BRD-A37630846: 相似 drug 数 1, 自身正样本 26, 候选难负样本 97

4. 含义
- 这些样本不一定要“删除”。
- 但它们非常适合作为“训练负样本去噪”的观察重点。


五、相似性矩阵观察
1. 已使用的相似性矩阵
- drug_rdkit.csv: 453 x 453
- drug_DGen_sim.csv: 453 x 453
- drug_ge_sim.csv: 453 x 453
- side_mesh_sim.csv: 1019 x 1019
- adr_GDisease_sim.csv: 1019 x 1019

2. 基本情况
- drug_rdkit, drug_DGen, side_mesh, adr_GDisease 都是对称矩阵，对角线为 1。
- drug_ge_sim 是对称矩阵，但存在负值，范围约为 [-0.3257, 1.0]。

3. 说明
- GE 相似度含负值本身不一定错误，但说明该特征和另外几类相似度的统计分布不同。
- 如果后续想进一步做数据优化，可以单独观察 GE 是否引入了更多不稳定样本。


六、哪些优化方式更有利于指标提升
以下建议按“更安全 -> 更激进”排序。

方案 A: 低支持实体清洗
目标:
- 移除极低支持的 drug / ADR，减少高噪声和高方差样本。

建议做法:
1. 尝试去掉正样本数 <= 1 的实体
2. 再尝试去掉正样本数 <= 2 的实体
3. 再尝试去掉正样本数 <= 3 或 <= 5 的实体

为什么有利:
- 单例和双例实体几乎没有稳定模式，最容易拖累 AUC/AUPR。
- 清洗后标签图更稳定，模型更容易学到规律。

风险:
- 这是在缩小任务定义，不是纯算法提升。
- 分数会更好看，但覆盖范围会变小。

建议:
- 必须把“原始版”和“清洗版”分开报告，不能混写结果。


方案 B: 训练负样本去噪
目标:
- 不动正样本，只减少明显可疑的伪负样本。

建议做法:
1. 对每个 drug，找结构相似度高于阈值的近邻 drug
2. 若近邻 drug 已知关联某个 ADR，而当前 drug 该位置是 0，
   则把这个 0 标为“可疑负样本”
3. 在训练时排除这类可疑负样本，不作为训练负样本

为什么有利:
- 这类样本最可能把本应靠近正样本的点错误推向负类
- 去噪后，AUC 和 AUPR 通常更容易提升

风险:
- 阈值选得太激进会把太多困难负样本删掉
- 如果测试集也一起删，会有明显刷分嫌疑

建议:
- 只对训练集负样本做去噪
- 测试集保持原样，保证指标可信


方案 C: 困难负样本降采样
目标:
- 不是删实体，而是改变训练时抽到哪些负样本

建议做法:
1. 维持正负样本 1:1 平衡
2. 不再完全随机抽负样本
3. 先剔除最可疑的伪负样本
4. 再从剩余负样本中采样

为什么有利:
- 当前负样本量远大于正样本
- 负样本质量比数量更重要

风险:
- 如果只保留太容易的负样本，训练会失真
- 应该保留一部分真正困难但可信的负样本


方案 D: 只删“不利于 AUC”的测试样本
结论:
- 不建议直接这样做

原因:
1. 这会直接改变评估任务
2. 得分会更高，但可信度会明显下降
3. 很难对外说明“提升来自模型”还是“提升来自删测试样本”

建议:
- 如果一定要研究这一方向，只能作为“观察实验”
- 不能把这种结果当成正式结果


七、最推荐的优化顺序
推荐顺序如下:

第 1 步:
- 先做“低支持实体统计版实验”
- 例如:
  - baseline: 不清洗
  - clean_v1: 去掉正样本数 <= 1 的 drug 和 ADR
  - clean_v2: 去掉正样本数 <= 2 的 drug 和 ADR

第 2 步:
- 做“训练负样本去噪版实验”
- 原则是:
  - 只改训练负样本
  - 测试集完全不动

第 3 步:
- 在去噪后再做负样本采样优化
- 比较:
  - 随机负采样
  - 去噪后随机负采样
  - 去噪后分层负采样


八、对指标提升最有希望的方向
如果目标是“既提升指标，又尽量保住结果可信度”，最推荐的是:

1. 训练负样本去噪
- 优先级最高
- 原因是当前负样本本来就不是严格真负样本

2. 低支持实体清洗
- 第二优先级
- 尤其是 ADR 侧，因为 ADR 单例和双例数量非常多

3. GE 特征稳定性复查
- 优先级第三
- 因为 GE 相似度分布和其他特征差异较大，可能对边界样本带来噪声


九、结论
当前数据集不是“没有优化空间”，而是“优化空间主要在标签噪声和极低支持样本”。

最重要的几个判断:
1. 当前大量负样本其实是未观察样本，不是严格真负样本
2. ADR 侧极低支持样本非常多，是拖累 AUC/AUPR 的高风险来源
3. 直接删除测试难样本虽然能提分，但不建议作为正式优化路线
4. 更合理、更稳妥的方向是:
   - 低支持实体清洗
   - 训练负样本去噪
   - 更干净的负采样策略


十、建议的下一步
建议下一步先不要直接改正式训练集，而是先做一组“观察版实验设计”:

1. baseline
- 保持当前数据不变

2. clean_support_v1
- 去掉正样本数 <= 1 的 drug 和 ADR

3. clean_support_v2
- 去掉正样本数 <= 2 的 drug 和 ADR

4. clean_negative_v1
- 仅对训练集排除“高相似邻居支持的可疑负样本”

5. 对比 AUC / AUPR / ACC / MCC
- baseline 和各 clean 版本单独记录
- 不能混写

本报告结论:
- 可以通过优化数据集来提升指标
- 但最推荐的是“去噪”和“低支持清洗”
- 不推荐直接为了 AUC 去删测试难样本
